Things I learned in JavaScript

Arrow functions are used to shorten javascript functions.

  • Always: Delete the word function, add "=>" after the parameters.
  • If the function has only one parameter: Delete the parentheses around the parameter.
  • Ps.: You cannot delete the parentheses if there are more than one parameter or if there are no parameters.
  • If the function has only one line of code: Delete the curly brackets and the "return".

Template literals are used to create strings with embedded expressions.

  • Always: Use backticks (`) instead of quotes (') or double quotes (").
  • To embed an expression: Use the syntax ${expression} inside the backticks.

Different types of data can be stored inside arrays.

  • In the example, some elements are strings, others are numbers, and the last one is boolean.

"For in" isn't as simple as I had thought.

  • - If you don't use brackets, you'll only get the keys (nome and idade, in this case)
  • If you use it in an array, it will return the indices of the array elements.
  • To get the values of an array, use "for of" instead of "for in".

Ternary operations are a shorthand way to write if/else statements.

  • It has the syntax: condition ? value_if_true : value_if_false
  • In this case, if number is perfectly equal to 2, "then" it returns "2", otherwise it returns "not 4"

The && and || operators

  • The && operator returns the first falsy value or the last value if all are truthy.
  • The || operator returns truthy if at least one operand is truthy.

The Switch Case statement

  • It's a more elegant and readable way to handle multiple conditions compared to using multiple if/elif statements.
  • It is similar to the "case" statement in Python.

Synchronous and Asynchronous Functions

  • Synchronous functions block execution until they finish.
  • Asynchronous functions do not block execution and allow other code to run while they are running.

JavaScript Libraries

  • Aprendi que existem centenas de milhares de bibliotecas no JavaScript, cada uma buscando sintetizar um código (uma ação). Em vez de digitar algo complexo em ECMAScript (vanilla JavaScript), você simplesmente usa uma biblioteca com uma linguagem mais acessível e ela depois transforma o código em vanilla js para você.
  • Simplesmente digitar "import axios from "axios";" não funcionaria. Antes é necessário instalar essa biblioteca com "npm install axios".
  • Ainda há outro passo para funcionar: Um "bundler" (Vite, Webpack, Parcel), pois o navegador não consegue ler essa ação.
  • Se não fosse utilizar esse meio, precisaria fazer à maneira antiga:

    • <script> src="https://cdn.jsdelivr.net/npm/axios/dist/axios.min.js"></script>
    <script>
    // Agora você não usa 'import', o axios já está disponível globalmente
    axios.get('https://api.github.com/repos/facebook/react');
    </script>

Bundlers

  • Bundlers are tools that combine multiple JavaScript files into a single file for efficient loading.
  • They help manage dependencies and optimize code for production environments.

Diferença entre biblioteca de utilitários e biblioteca de arquitetura

Difference between library and framework

  • Axios: Biblioteca de utilitários - Focado em um tarefa específica
  • React (e outros): Biblioteca de arquitetura - Ditam como o site é construído/interface
  • Importante: No final das contas, todas as bibliotecas conversam entre si porque seus padrões são traduzidos para ECMAScript (vanilla js).

Then and Catch

  • Quando usamos Axios, o algoritmo é simples: se a requisição for bem-sucedida, o then é chamado; se falhar, o catch é chamado.
  • Sem o Axios, precisaríamos lidar com Promises diretamente, usando then() e catch() para tratar sucesso e falha.
  • Pode-se se usar apenas "then" tanto para sucesso quanto para falha, mas o catch é mais comum para tratar erros.

Constructor

  • Constructors are special functions that are used to create and initialize objects of a class.
  • They are called when an object of the class is created.

This

  • "This" is a keyword in JavaScript that refers to the current object context.
  • Its value depends on how a function is called, and it can refer to different objects in different contexts.
  • The only purpose of using "this" is to avoid global namespace pollution.

JavaScript DOM

  • The Document Object Model (DOM) is a programming interface for HTML and XML documents.
  • It represents the structure of a document and allows scripts to dynamically access and update the content, structure, and style of the document.
  • JavaScript can interact with the DOM to manipulate elements, handle events, and create dynamic content.

Selecting Elements

  • JavaScript provides several methods to select elements from the DOM, such as getElementById(), getElementsByClassName(), and querySelector().
  • These methods allow developers to target specific elements for manipulation or event handling.
  • While getElementById() is the fastest, querySelector() is the most flexible.
  • However, sometimes it can be necessary to use querySelectorAll() to select multiple elements.

Events

  • Events are actions that occur in the browser, such as clicking a button or submitting a form.
  • JavaScript can respond to these events using event listeners, which allow developers to execute code when an event occurs.
  • Common event types include click, hover, keydown, and submit.
  • Event listeners can be added using the addEventListener() method.

Click Event

  • A click event is triggered when a user clicks on an element, such as a button or a link.
  • JavaScript can listen for click events and execute code in response, allowing for interactive web pages.
  • Click events can be used to toggle visibility, submit forms, or navigate to different pages.
  • The letter "e" in the event handler function parameter represents the event object, which contains information about the event. However, any variable name can be used.
